# Instinct: Commercial Quality Standard

## The Pattern
Every output should be production-ready.

## When It Fires
- Writing any code.
- Writing documentation.
- Writing a commit message.
- Creating a configuration file.

## Correct Behavior
- Code has error handling, types, and docstrings.
- Commit message has scope and body following Conventional Commits.
- README has usage examples and prerequisites.
- Configuration files are properly commented.

## Failure Mode
- "TODO: add error handling" left in the code.
- Placeholder text left in documentation.
- Commit message: "fix stuff".

## Override Protocol
User explicitly says "rough draft is fine", "just sketch it out", or "wip commit".
